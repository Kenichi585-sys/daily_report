1. HTMLフォームの解剖（見た目と意味）
- 紐付けのルール: `<label for="A"> と <input id="A"> `を同じ名前にすることで、文字をクリックしても入力欄にフォーカスが当たる
- Bootstrap: クラス名（form-controlなど）をつけることで、ただのHTMLタグがリッチなデザインになる。

2. フォームとバックエンド（Django）の連携

- 名前の一致: HTML側の name="price" と、Views側の fields = ['price'] が一致していないと、データは無視される
  - 送信の3種の神器:
    - method="post": データを封筒に入れて隠して送る（登録処理の基本）。
    - {% csrf_token %}: 偽サイトからの攻撃を防ぐセキュリティ手形。
    - enctype="multipart/form-data": 画像のような重いデータを「別の箱」に入れて送るための呪文。
    
3. UX（ユーザー体験）の考慮
- 数値入力: type="number" や step 属性を使うことで、スマホで数字キーボードを出したり、入力を制限したりできる。

4. 静的ファイルとデバッグ
- 画像の出し分け: テンプレート内で {% if product.image %} を使い、画像がある時とない時（NO IMAGE）を切り替えるロジック。
- Staticファイルのパス: {% load static %} と {% static 'パス' %} を使うのがDjangoの作法。
