e.preventDefault()について
送信ボタンを押すと、ブラウザは HTTP リクエストを送り、返ってきた HTML でページ全体を描き直す — これがフォームのデフォルト動作です。昔の Web アプリはこれが普通でした。
1. ユーザーが「作成」ボタンをクリック
2. ブラウザ: 「form の submit イベントが起きた」
3. React: handleSubmit が実行される
4. ブラウザ: デフォルト動作としてフォーム送信を実行
   → ページ遷移 / リロード

e.preventDefault() を 3と4の間 で呼ぶと、4のデフォルト送信を止められます。
