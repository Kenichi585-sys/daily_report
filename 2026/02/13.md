### わかったこと
- JSカレンダーブランチに戻り、Gitの起点を空のメインに合わせるため以下のコマンドを実行した。
    - git reset –soft main
- その後は `git add`, `git commit` をして最後にプッシュをしたのだが、その時は強制的にプッシュをする必要があったため、`--force` というコマンドを一番最後につけた。`--force` のコマンドは "-f", 実際に入力実行したコマンドは以下
    - git push origin js_calendar -f
- `git reset --soft main`
    - これを打つとフォルダの中身はそのままで、Gitの起点だけ空っぽのメインと同じ地点まで戻るという形になり、コミットだけ取り消した状態にできる。この状態で強制的にプッシュした `js-calendar` ブランチは、まだコミットされていない新規追加ファイルとしてプッシュができた。
