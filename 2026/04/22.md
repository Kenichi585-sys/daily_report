### わかったこと
- オプショナルプロパティとオプショナルチェーンという2つの概念がある。
- オプショナルプロパティ
    - プロパティ定義の際に「?」をつけるもの。
    - (a) 今回のTypeScriptの課題でいうと、クーポンコードがある場合とない場合の両方が考えられる。
    - (b) 例えば、interface CartItem の定義で couponCode?: string のように記述する。
    - (c) これにより「このプロパティは存在しないこともある」という意味になる。型定義は string だが、実際には string | undefined という形で undefined の可能性も含まれる。
- オプショナルチェーン
    - プロパティへのアクセス時に「?.」をつけるもの。
    - (a) 左側の値が null または undefined だった場合、そこで処理を止めて undefined を返す。
    - (b) 例えば state.activeCoupon?.code と書けば、activeCoupon が null であってもエラーにならずにアクセスできる。
- 今回、課題で使用したのはプロパティとして記述する方だった。プロパティ（キー）として書くクーポンコードに「?」をつけるので、これはオプショナルプロパティに該当する。

- 「プロパティ定義なのか」「アクセス時なのか」という違いはあるけど、どちらも「値がある場合とない場合の両方を考慮している」という本質的な部分は同じ
